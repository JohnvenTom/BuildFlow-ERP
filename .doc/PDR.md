1 文档概述
1.1 文档目的
本文档为项目标准 PRD，完整定义建材进销存 CRM 系统全部业务需求、页面功能、交互逻辑、业务规则。作为后端 SpringBoot 开发、前端 Vue 页面开发、数据库设计、测试用例编写、实施交付的统一依据，同时满足作业 70 页全套文档配套需求。
1.2 文档范围
覆盖系统全部五大核心业务模块、数据看板、登录权限、通用基础功能；不含底层 SpringBoot 技术实现细节、服务器运维配置。
1.3 读者对象
产品、后端 Java 开发、前端开发、测试工程师、实施顾问、项目评审人员。
1.4 术语与缩略词
CRM：客户关系管理，统一管理客户、供应商档案与跟进记录
WMS：仓储管理，多仓库、批次、色号库存管控
SpringBoot：后端 Java 企业级开发框架，提供事务、权限、REST 接口
CRUD：新增、查询、编辑、删除基础页面功能
应收：客户未结算销售货款
应付：企业欠供应商采购货款
工地单：建材行业特有分次配送出库单据
呆滞库存：连续 90 天无出库的积压建材
2 项目背景与目标
2.1 行业现状与业务痛点
本系统面向瓷砖、管材、防水、砂石、门窗等区域中型建材流通贸易商，当前线下经营存在五大核心痛点：
客户分散（装修公司、工程项目、分销商、散户），无统一电子档案，销售跟进记录零散，客户信用、欠款无统一管控，坏账率高；
采购、送货全部纸质单据，人工记账，多仓库库存账实不符，建材色号、生产批次无法追溯；
工地订单分批送货，手工统计工作量大，无法自动核算业务员销售提成；
财务每月手工汇总采购、销售单据制作应收应付台账，对账效率极低；
总部无法实时查看各门店、工地销量、库存积压、逾期欠款，经营决策缺少数据支撑；
廉价单机进销存无多组织、工地配送、CRM 一体化能力，无法满足连锁建材企业数字化需求。
2.2 项目建设目标
搭建统一数字化中台，整合 CRM、采购、销售、仓储、财务五大业务，全流程线上单据流转；
建立完整客户供应商生命周期管理，自动信用管控、逾期回款预警；
库存全链路追溯，采购入库、工地出库、调拨、盘点自动更新库存，Spring 事务保证数据一致；
自动生成应收、应付台账，减少财务 80% 手工对账工作；
总部可视化经营大屏，实时展示销量、库存、欠款、客户数据；
系统全部页面实现完整 CRUD 操作，可本地部署、完整流程演示，满足企业级交付标准。
2.3 项目商业价值与定价依据
市场规模
国内建材流通行业市场规模超 5 万亿，区域中型经销商数字化改造渗透率不足 30%，一体化进销存 CRM 定制系统市场主流报价区间 120 万 - 280 万。
项目总报价：195 万元
SpringBoot+Vue 标准化软件授权：85 万
建材行业定制开发（工地分批送货、色号批次、提成、财务对接）：70 万
现场实施、历史数据迁移、全员培训：20 万
一年原厂运维、版本迭代、BUG 修复：20 万
定价优势
采用 Java SpringBoot 企业级架构，支持数据库事务保障财务、库存数据安全；支持多门店、多工地、多仓库分布式管理；配套数据大屏、移动端开单、自动预警等高价值功能，区别于低成本单机进销存软件。
3 业务整体概述
3.1 业务主体
上游：建材供应商（瓷砖厂、管材工厂、砂石供货商）
企业内部：总部、线下门店、仓库管理员、销售业务员、财务、管理员
下游客户：装修公司、工程项目工地、区域分销商、零售散户
3.2 全业务主流程总图
供应商→采购订单→采购入库（库存增加）→门店 / 工地销售开单→分次出库（库存减少）→销售退货（库存回加）→仓库调拨 / 盘点修正库存→自动生成应收应付台账→回款 / 付款核销往来款。
3.3 用户角色划分
系统管理员：全部模块增删改查、账号权限配置、基础档案维护
销售业务员：客户管理、销售开单、查看自身订单、客户跟进记录
仓管人员：商品、仓库管理、入库、出库、调拨、盘点操作
财务人员：应收应付台账、回款付款登记、报表导出、对账
只读查看员：仅查询数据，无新增、编辑、删除权限
4 全局通用需求
4.1 登录与权限体系
账号密码登录，密码加密存储，连续 5 次输错锁定账号 10 分钟；
基于 SpringSecurity 实现角色菜单权限控制，不同角色仅展示对应功能菜单；
支持账号新增、禁用、角色分配、密码重置 CRUD 页面；
操作日志：所有新增 / 编辑 / 删除单据记录操作人、操作时间、操作内容。
4.2 通用页面规范
所有列表页面统一：搜索条件区 + 数据表格 + 分页控件 + 新增 / 导出 / 打印按钮；
弹窗表单统一布局，必填项标红，提交前前端 + 后端双重参数校验；
分页统一参数 pageNum、pageSize，后端 Spring 接口统一分页返回结构；
删除统一二次弹窗确认，重要单据支持作废而非物理删除（逻辑删除 is\_deleted=1）。
4.3 单据通用规则
所有单据自动生成唯一单据编号，编码规则：业务类型 + 年月日 + 流水号；
单据状态：草稿、已审核、已作废；草稿可编辑，审核后仅允许作废；
单据修改 / 作废自动触发库存、应收应付数据回滚（Spring 事务保证原子性）。
4.4 通用数据字典
系统内置固定字典，后端统一接口提供前端下拉选择：
客户类型：工程项目、装修公司、分销商、零售散户
商品分类：瓷砖、管材、防水材料、门窗、砂石
单据状态：草稿、已审核、已作废
往来款状态：正常、逾期、已结清
仓库类型：总部中心仓、门店仓、工地临时仓
4.5 导出打印功能
所有列表、单据支持：Excel 导出、PDF 预览、纸质打印；打印模板适配建材送货单、对账单格式。
5 核心模块详细需求
5.1 模块一：CRM 客户供应商管理
模块目标
统一管理上游供应商、下游全类型客户，记录销售跟进，管控信用额度与逾期欠款。
页面 1：客户档案管理（CRUD 完整）
新增客户：客户名称、客户类型、工地地址、联系人、联系电话、信用额度、业务员标签；
编辑客户：修改基础信息、调整信用额度；
删除客户：仅未产生订单的空白客户可删除，有业务单据仅逻辑作废；
查询筛选：客户名称、类型、业务员、欠款状态、关键词模糊搜索；
列表展示：客户名称、类型、地址、信用额度、当前未结欠款、负责业务员。
页面 2：供应商档案管理（CRUD 完整）
字段：供应商名称、主营品类、联系人、电话、结算周期、应付上限、合作状态；支持新增、编辑、查询、作废。
页面 3：客户跟进记录台账
新增跟进：关联客户、跟进类型（电话 / 上门 / 报价）、跟进内容、下次回访时间；
编辑、删除、分页查询；
筛选：客户、时间段、跟进业务员。
页面 4：客户信用 & 逾期欠款查询
展示所有客户当前应收总额、信用剩余额度、逾期金额、逾期天数；
支持一键导出逾期催款报表。
核心业务规则
客户创建销售订单时自动校验：订单金额超出剩余信用额度禁止保存；
客户回款单保存后自动扣减对应应收欠款，更新可用信用额度；
跟进记录与客户强关联，删除客户同步清理跟进记录。
5.2 模块二：采购管理模块
模块目标
管理向供应商进货全流程，采购订单、入库、退货，自动增加仓库库存。
页面 1：采购订单 CRUD
新建订单：选择供应商、商品明细（品类、规格、色号、采购单价、数量）；
草稿保存、提交审核；审核后可生成采购入库单；
查询：供应商、单据日期、订单状态筛选。
页面 2：采购入库单 CRUD
可直接引用采购订单一键带入商品明细；
选择入库仓库，填写实际入库数量；
审核入库后系统自动增加对应仓库商品库存（Spring 事务，入库失败全部回滚）。
页面 3：供应商退货单
选择供应商、原入库单据、退货商品、退货数量；
审核后扣减库存，同步减少应付账款。
页面 4：月度采购对账单
自动汇总时间段内采购、退货金额，生成对账单支持导出 PDF 发给供应商对账。
业务规则
未审核采购订单可随意编辑；审核后仅能通过退货单冲抵；
入库、退货操作同时更新库存与供应商应付台账，数据变更事务绑定。
5.3 模块三：销售与工地出库管理（核心营收模块）
模块目标
实现工地、门店销售开单，支持分次配送出库，自动核算业务员提成，出库扣减库存。
页面 1：销售订单 CRUD
选择客户、业务员、商品明细（规格、色号、销售单价、数量、折扣）；
自动计算订单总金额、该业务员提成金额；
草稿保存、提交审核；支持按工地分批送货。
页面 2：工地分次出库单
关联销售订单，选择本次配送商品及出库数量；
多次出库后系统记录订单剩余未发货数量；
出库审核成功，扣减对应仓库库存，同步增加客户应收款。
页面 3：销售退货单
客户退回商品，选择原出库单，录入退货数量；审核后返还库存、扣减客户应收。
页面 4：业务员提成核算报表
按时间段汇总每个业务员全部订单提成金额，支持导出核算工资。
业务规则
出库校验库存不足则拦截单据提交，提示库存短缺；
同一销售订单允许多次生成出库单，直至全部商品发货完毕；
订单、出库、退货全部自动同步财务应收台账。
5.4 模块四：WMS 仓储库存管理
模块目标
多仓库统一管控，商品档案、调拨、盘点、库存预警，保证账实同步。
页面 1：商品档案 CRUD
建材专属字段：商品名称、分类、规格、色号、单位、采购价、销售价、最低库存预警值。
页面 2：仓库档案 CRUD
仓库名称、仓库类型、地址、管理员，支持多仓库创建维护。
页面 3：仓库调拨单
调出仓库、调入仓库、调拨商品数量；
审核后，调出仓库存减少，调入仓库存增加，事务保证两边同时变更。
页面 4：库存盘点单
导出盘点空白表线下清点，回填系统实盘数量；
系统自动计算盘盈 / 盘亏，审核后修正账面库存。
页面 5：库存预警报表
低库存：当前库存低于预警值商品；
呆滞库存：90 天无任何出库记录积压建材；
支持导出预警清单采购补货。
5.5 模块五：应收应付财务对账模块
模块目标
自动汇总业务单据生成往来台账，登记回款、付款，监控逾期款项。
页面 1：客户应收台账
按客户汇总所有未结清销售出库总金额、已回款金额、当前欠款、逾期天数。
页面 2：供应商应付台账
汇总所有采购入库未支付货款，展示每家供应商待付款总额。
页面 3：客户回款单 CRUD
选择客户、抵扣对应销售欠款，录入回款金额、付款方式；保存后自动减少应收。
页面 4：供应商付款单 CRUD
向供应商支付货款，抵扣应付账款。
页面 5：逾期欠款预警列表
筛选所有产生逾期的客户，支持批量导出催款单据。
业务规则
应收应付全部由采购、出库、退货单据自动生成，禁止财务手工新增往来；
回款 / 付款单据审核后永久留存，作为财务对账原始凭证。
6 数据可视化大屏需求
系统首页经营大屏，无需 CRUD 查询功能，仅数据展示，后端统一聚合统计接口：
顶部核心指标：今日销售额、本月总销量、总应收欠款、呆滞库存数量；
分平台客户销量柱状图：工程项目、装修公司、分销商销售占比；
商品热销排行 TOP10；
逾期欠款客户榜单；
近 30 日销售额趋势折线图；
各仓库当前库存总量饼图。
7 非功能性需求
7.1 性能需求
列表分页查询响应时间≤300ms（SpringBoot 后端接口）；
单据保存、库存变更事务处理≤500ms；
数据大屏聚合查询≤1s；
单页面列表单次加载最多 100 条，大数据分页加载，禁止一次性全量查询。
7.2 兼容性需求
PC 端：Chrome、Edge 主流浏览器；
后端 SpringBoot 打包 Jar，支持 Windows、Linux 本地运行；
数据库兼容 MySQL8.0。
7.3 安全需求
用户密码 MD5 加盐加密存储；
后端接口统一登录 token 鉴权，未登录禁止访问任何业务接口；
所有删除、作废、资金类操作记录完整操作日志；
SQL 参数化，SpringMyBatis 防止注入攻击。
7.4 可扩展性需求
商品、客户、单据预留扩展字段，后期可快速增加业务属性；
SpringBoot 分层架构，新增模块可独立开发，不改动原有核心业务代码；
预留第三方财务软件对接 API 接口，支持后期拓展集成。
8 交付与演示要求
前端所有模块页面完整 CRUD 交互，无功能缺失；
SpringBoot 后端接口全部可调试，数据库事务逻辑完整；
提供初始化测试数据，开箱即可演示采购、销售、库存、财务全流程；
提供 3-5 分钟录屏演示视频，完整走通一套业务闭环；
配套数据库设计、API 接口、部署手册全套文档。
